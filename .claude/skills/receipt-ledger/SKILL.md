# 영수증 회계장부 만들기 (receipt-ledger)

영수증 이미지 묶음을 받아 **카테고리별 폴더 정리 + 회계장부 엑셀**을 한 번에 생성한다.
원본은 절대 건드리지 않고, 결과는 지정한 `Output` 폴더 안에 모은다.

## 언제 사용하나

- "이 영수증들 정리해서 회계장부 엑셀 만들어줘"
- "영수증 카테고리별로 폴더 나눠줘"
- M07 실습(영수증 자동 정리 Skill) 재실행

## 결과물

```
<Output>/
├── 식비/          2026-06-02_하늘카페 강남점.png ...
├── 도서비/        ...
├── 사무용품/      ...
├── 기타/          ...
└── 회계장부_결과.xlsx   (영수증입력 · 회계장부 · 카테고리별집계 3시트)
```

## 실행 절차

### 1) 영수증 이미지 읽기 (에이전트가 직접 수행)

대상 폴더의 각 영수증 이미지를 Read 도구로 열어 다음을 추출한다.

| 항목 | 설명 |
|---|---|
| `date` | 거래일시 → `YYYY-MM-DD` |
| `merchant` | 상호(가맹점명) |
| `amount` | **합계** 금액(정수, 부가세 포함 최종액) |
| `payment` | 결제수단 (현금/신용카드/체크카드/간편결제 등) |
| `category` | 아래 기준으로 분류 |
| `memo` | 비고 — 대표 품목 요약 (예: "카페라떼 외") |
| `desc` | 회계장부 적요 보조설명 (예: "카페·디저트") |
| `file` | 원본 파일명 (사본 정리용) |

**카테고리 분류 기준**

- `식비` — 식당·카페·베이커리·편의점 식품·마트 식료품
- `교통비` — 지하철·버스·택시·KTX·주유
- `도서비` — 서점·도서·전공서적
- `사무용품` — 문구·프린터·컴퓨터 주변기기·전자 소모품
- `기타` — 위에 안 맞는 것(약국·의료·생활용품 등)

애매하면 대표 품목의 성격으로 판단하고, 판단 근거를 사용자에게 간단히 보고한다.

### 2) 입력 JSON 작성

추출 결과를 `input.json`(작업용 임시 폴더)에 저장한다.

```json
{
  "source_dir": "<원본 영수증 폴더 절대경로>",
  "output_dir": "<Output 폴더 절대경로>",
  "output_name": "회계장부_결과.xlsx",
  "opening_balance": 500000,
  "opening_date": "2026-06-01",
  "receipts": [
    {"file":"receipt_ko_01.png","date":"2026-06-02","merchant":"하늘카페 강남점",
     "amount":28100,"payment":"현금","category":"식비",
     "memo":"카페라떼 외","desc":"카페·디저트"}
  ]
}
```

- `opening_balance`(기초잔액)·`opening_date`는 선택. 기본 500,000원 / 가장 이른 영수증 날짜.
- `file`이 없으면 폴더 정리는 건너뛰고 엑셀만 만든다.

### 3) 스크립트 실행

```bash
python "<이 스킬 폴더>/build_ledger.py" "<input.json 경로>"
```

스크립트가 자동으로:
1. `Output/<카테고리>/`에 `날짜_가맹점.확장자` 이름으로 **사본** 저장(원본 보존)
2. 3시트 엑셀 생성 — 서식·수식(잔액 누계, SUMIF 집계, 비율%)까지 양식과 동일
3. 처리 건수·카테고리별 집계·잔액을 콘솔에 요약 출력

### 4) 검증

- 엑셀 `회계장부` 시트의 **합계 차변** = 영수증 합계 총액인지 확인
- `카테고리별집계` 비율 합계 100% 확인
- 다른 영수증 묶음(예: 테스트용)으로 재실행해도 동작하는지 확인

## 의존성

```bash
python -m pip install openpyxl
```

## 참고

- 엑셀 잔액/집계는 **수식**으로 들어가므로 값 수정 시 자동 재계산된다.
- 카테고리별집계 시트에는 실제 등장한 카테고리만 표기된다(교통비가 없으면 생략).
